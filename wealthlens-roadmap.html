Build Phases
1
Foundation
Auth + Plaid + Supabase Schema
Week 1–2
▶

Stand up the iOS app skeleton in SwiftUI with Supabase Auth (Sign in with Apple + email). Integrate Plaid Link so users can connect real bank accounts. Design the Postgres schema with Row Level Security locked from day one — every user only ever sees their own data.

🏗️
SwiftUI app skeleton
TabView, NavigationStack, async/await data layer, Supabase Swift SDK
🔐
Sign in with Apple
Supabase Auth + Apple OAuth, JWT refresh, Keychain token storage
🏦
Plaid Link integration
Link token flow, exchange public_token, store access_token encrypted in Edge Fn vault
🗄️
RLS schema
users, plaid_items, accounts, transactions — every table locked with auth.uid() policies
Supabase features used
Auth (Apple OAuth)
Row Level Security
Postgres schema migrations
Vault (secrets)
✓ Milestone: User signs in → connects Chase account → transactions appear in DB with RLS enforced
2
Realtime
Live Balance Updates + Transaction Feed
Week 3
▶

Wire Supabase Realtime to the transactions table. When Plaid webhooks fire (new transaction, balance refresh), an Edge Function updates the DB and the iOS app reflects the change instantly — no polling, no pull-to-refresh. The feed feels alive.

⚡
Plaid webhook Edge Fn
Deno function receives Plaid webhooks, validates signature, upserts transactions to Postgres
📡
Supabase Realtime sub
Subscribe to postgres_changes on transactions table — SwiftUI @Observable auto-updates
📊
Transaction feed UI
Merchant logo, category badge, amount, animated row insert on live update
💳
Balance refresh
Daily Supabase cron (pg_cron) → Edge Fn → Plaid balance fetch → realtime push to iOS
Supabase features used
Realtime (postgres_changes) ★
Edge Functions (Deno)
pg_cron (scheduled jobs)
✓ Milestone: Make a purchase on linked card → transaction appears in iOS app feed within 5 seconds, no user action
3
AI Search
pgvector Semantic Search + Chat
Week 4–5
▶

This is the killer feature. Every transaction gets embedded via OpenAI text-embedding-3-small and stored in a pgvector column. A chat interface lets users ask anything: "how much did I spend on coffee in January?", "find all Uber charges over $30", "what's my biggest spending category this quarter?" — answered by vector similarity + SQL aggregation, then narrated by Claude/GPT.

🧠
Embedding pipeline
Edge Fn triggered on transaction insert → OpenAI embed → store vector[1536] in pgvector column
🔍
Semantic search RPC
Postgres function: embed query → cosine similarity search → return top-k transactions
💬
Chat UI (SwiftUI)
Bubble chat, streamed LLM response, transaction card attachments in thread
🤖
Finance agent (Edge Fn)
System prompt with user's financial context → vector search results → GPT-4o narration
Supabase features used
pgvector extension ★
match_transactions() RPC
Edge Fn streaming
HNSW index (fast ANN)
✓ Milestone: Type "Did I overspend on food last month?" → AI answers with exact amounts and top merchants
4
Intelligence
Budgets, Goals + Proactive Nudges
Week 6–8
▶

Turn passive tracking into active guidance. Users set budgets per category and savings goals with a target date. Supabase pg_cron runs nightly analysis — if you're 80% through your food budget by mid-month, you get a push notification. Goal progress is tracked in realtime as transactions come in.

🎯
Budget engine
Category budgets table, monthly rollup via Postgres views, percentage tracking
📈
Savings goals
Target amount + date → auto-calculates required monthly save rate, progress bar
🔔
Smart push notifications
pg_cron nightly job → Edge Fn → APNs via Expo push tokens → budget alerts
📉
Insights dashboard
Monthly spend by category chart, month-over-month delta, recurring subscriptions detector
Supabase features used
Postgres Views (rollups)
pg_cron (nightly analysis)
Realtime (goal progress)
Edge Fn (APNs delivery)
✓ Milestone: User hits 80% of restaurant budget → push notification fires before they overspend

Launch
TestFlight Beta + App Store + Monetization
Week 9–10

Ship it. TestFlight beta to 100 users, iterate on feedback, then App Store submission. Monetize via freemium — free tier has 1 connected account + basic chat, Pro unlocks unlimited accounts, full AI chat history, and advanced analytics. Use RevenueCat for subscription management.
